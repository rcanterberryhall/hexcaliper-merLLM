Configuration

Day / Night Mode

Day mode runs two Ollama instances simultaneously — one per GPU — and routes requests by model name. The model assigned to GPU 1 (DAY_MODEL_GPU1) goes to that instance; everything else goes to GPU 0.

Night mode activates automatically when both conditions are met: no interactive requests for INACTIVITY_TIMEOUT_MIN minutes and the current time is past the schedule hint (BASE_DAY_END_LOCAL, optionally adjusted for your timezone via GeoIP). In night mode, merLLM switches to a dual-GPU Ollama instance and drains the batch queue with extended context (NIGHT_NUM_CTX).

Any interactive request during night mode triggers an immediate return to day mode.

Manual override — use the Day / Night / Auto buttons on the Overview tab. Auto clears the override and returns to schedule-driven behaviour.

Batch Jobs

Batch jobs are generation requests that run during night mode with a larger context window. They're useful for deep-analysis tasks that would block interactive use during the day.

Submit a job from the Batch Jobs tab or via the API: 

POST /api/batch/submit
{
  "source_app": "parsival",
  "prompt": "Analyze...",
  "model": "qwen3:32b"   // optional
}

GET  /api/batch/status/<id>
GET  /api/batch/results/<id>
POST /api/batch/<id>/cancel
POST /api/batch/<id>/requeue

Jobs stay in queued status until night mode activates, then run sequentially. Failed jobs can be requeued; cancelled jobs can only be resubmitted.

Ollama Proxy

merLLM is a drop-in OLLAMA_BASE_URL replacement. Point LanceLLMot and Parsival at http://<host>:11400 and they require no other changes.

Supported endpoints:

POST /api/generateRouted by model name
POST /api/chatSame routing
POST /api/embeddingsAlways GPU 0
GET /api/tagsModel list (GPU 0)
POST /api/showGPU 0
POST /api/pullGPU 0 (shared store)
GET /api/psAggregated from both

Add X-Priority: batch to a request to place it in the low-priority queue without submitting a formal batch job.

Settings

All settings can be changed live from the Settings tab — no restart required. Changes are persisted in SQLite and applied immediately to the running process.

Key settings:

SSH Terminal

The SSH tab provides a full browser terminal via xterm.js. To enable it:

1. Set SSH_USER in docker-compose.yml (or via Settings).
2. Place your private key at ./data/ssh_key on the host (mounted as /data/ssh_key inside the container).
3. The key must be accepted by the target host's authorized_keys.

The connection is authenticated by SSH key — no passwords are stored or transmitted. Access is gated by Cloudflare Access at the nginx layer when deployed behind a tunnel.

VNC Viewer

The VNC tab proxies a WebSocket connection to the configured VNC server. To use it:

1. Start a VNC server on the host (e.g. TigerVNC or x11vnc).
2. Set VNC_HOST and VNC_PORT in settings if different from defaults (host.docker.internal:5900).
3. noVNC's rfb.js must be available — either served from the host or loaded via CDN. Without it the viewer shows a placeholder message.

merLLM acts only as a WebSocket-to-TCP proxy; it does not start or manage the VNC server.

GeoIP Scheduling

When a MaxMind GeoLite2-City database is present at GEOIP_DB_PATH, merLLM uses the client's IP address to determine their local timezone and adjusts the day-end schedule hint accordingly.

Without the database, BASE_DAY_END_LOCAL is used as-is with no adjustment.

To override the timezone lookup entirely, set GEOIP_OFFSET_OVERRIDE to a UTC offset (e.g. -5 for EST, 5.5 for IST).

The effective adjusted schedule is shown in the Overview tab status area and is also available from GET /api/merllm/geoip.
